# Getting Started

This section covers everything you need to understand and deploy OpenClaw Enterprise.

---

## In This Section

### [What Is OpenClaw Enterprise?](what-is-openclaw.md)

Understand what OpenClaw is, how OpenClaw Enterprise extends it, and the core principles that guide the project. Start here if you are new to the project.

### [Architecture Overview](architecture.md)

Learn how the system is structured: the OpenClaw Gateway, enterprise plugins, OPA policy sidecar, connectors, storage, and the Kubernetes operator. Includes data flow diagrams and the plugin dependency graph.

### [Key Concepts](concepts.md)

Reference page for the foundational concepts you will encounter throughout the documentation: plugins, skills, policy hierarchy, graduated autonomy, data classification, connectors, OCIP, audit trails, and fail-closed behavior.

### [Quickstart Guide](quickstart.md)

Deploy OpenClaw Enterprise on Kubernetes in six steps. Covers prerequisites, operator installation, database and connector configuration, instance deployment, policy creation, and verification.

---

## Recommended Reading Order

If you are evaluating OpenClaw Enterprise or setting it up for the first time, read the pages in the order listed above:

1. **What Is OpenClaw Enterprise** -- understand the value proposition and principles
2. **Architecture Overview** -- see how the pieces fit together
3. **Key Concepts** -- learn the vocabulary
4. **Quickstart Guide** -- deploy a working instance

Once you have a running deployment, proceed to the [User Guide](../user-guide/index.md) for day-to-day usage or the [Admin Guide](../admin-guide/index.md) for policy management and system administration.
